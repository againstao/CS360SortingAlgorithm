Sorting Made Simple

Selection Sorting can be described in a simple few points. Selection Link
    Simple Sort
  • Worst Case Runtime: O(n2)
  • Method of Sorting: Divides list into two parts, then compares adjacent elements and swaps them if needed.
  • This is a Stable, In-Place Sorting Algorithm
  • Worst Case Space Complexity: O(1) additional memory space
  • One of the least efficient sorting algorithms, but generally best for small lists
Insertion Sorting can be described in a simple few points. Insertion link
    Simple Sort
  • Worst Case Runtime: O(n2)
  • Method of Sorting: Reads one element each repitition, and then creates a sorted list out of the pulled elements
  • This is a Stable, In-Place Sorting Algorithm
  • Worst Case Space Complexity: O(1) additional memory space
  • Best for small data sets and is more efficient than Bubble and Selection sorts
Merge Sorting can be described in a simple few points. Merge Link
    Efficient Sort
  • Worst Case Runtime: O(n log n)
  • Method of Sorting: Divides list into n segments, then compares segments and merges them into one ordered list
  • This is a Stable, but not in-place sorting algorithm
  • Worst Case Space Complexity: O(n) additional memory space, but can be O(1) with a linked list
  • Best used for a linked list, but is not as efficient as the quicksort
Heap Sorting can be described in a simple few points. Heap link
    Efficient Sort
  • Worst Case Runtime: O(n log n)
  • Method of Sorting:Improved Selection sort. Divides the list into a sorted and unsorted area, and iteratively decreases the unsorted area by taking out the largest element and moving it to the sorted area
  • This is a Stable, but not in-place sorting algorithm
  • Worst Case Space Complexity: O(n) additional memory space
  • A little slower than the quicksort, but it has the advantage of having a O(n log n) runtime
Bubble Sorting can be described in a simple few points. Bubble link
    Simple Sort
  • Worst Case Runtime: O(n2)
  • Method of Sorting: Repeatedly steps through a list, comparing adjacent elements and swapping them if they are in the wrong order.
  • This is a Stable, In-Place Sorting Algorithm
  • Worst Case Space Complexity: O(1) additional memory space
  • One of the least efficient sorts, unless the input is mostly ordered with some elements out of order
Quick Sorting can be described in a simple few points. Quick link
    Efficient Sort
  • Worst Case Runtime: O(n2)
  • Method of Sorting: Comparison sort that divides a large array into two smaller sub-arrays and then recursively sort the sub-arrays
  • This is a Stable, but not in-place sorting algorithm
  • Worst Case Space Complexity: O(n), with the best being O( n log n)
  • About two or three times faster than the merge and heapsort. Similar to selection sort, but it does not always choose the worst-case partition
